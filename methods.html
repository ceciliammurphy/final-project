Methods

This project explores how social media analytics shape the way users interpret engagement, visibility, and performance online. The goal of Trulytics is to demonstrate how clearer data presentation can help users better understand patterns in their content and audience behavior. To create this prototype, I combined exported Instagram data with manually collected interaction metrics and web development tools to demonstrate how a more transparent analytics platform might function.

Data Collection

To build the prototype, I exported my personal Instagram data through Meta’s data download tool. I selected categories that provided structural insight into account activity, including followers, following, and post data. These datasets include timestamps and account relationships, which allow for analysis of growth patterns and posting behavior.

Because Instagram exports do not include usable post-level interaction totals, I manually collected interaction data from Instagram’s interface. This included likes, comments, and sends for recent posts. This allowed the project to incorporate a more complete representation of engagement rather than relying only on exported data.

Data Preparation

After exporting the data, I reviewed the JSON files to identify which categories were most useful for building the dashboard. The data was simplified and reorganized into smaller datasets, including follower growth, following growth, posting activity, and post interaction totals.

Follower and following data were grouped by timestamp to create growth visualizations over time. Post data was used to track posting frequency, while manually collected interaction totals were organized to compare engagement across recent posts. The goal was not to replicate Instagram’s exact analytics system, but to demonstrate how data could be structured more clearly.

Prototype Development

The Trulytics prototype was created using HTML, CSS, and JavaScript in Visual Studio Code. HTML structured the content, CSS controlled layout and design, and JavaScript was used to process data and generate charts through Chart.js.

The dashboard was designed as a series of data modules, including follower growth, following growth, posting activity, and post engagement. Each section presents a different aspect of performance while contributing to a larger understanding of user behavior and audience response.

Design Decisions

The visual design of Trulytics was influenced by existing social media interfaces, particularly Instagram’s layout and color palette. The goal was to create a familiar, minimal interface that keeps the focus on the data. Card-based sections, consistent spacing, and clear headings were used to improve readability.

Design choices prioritize clarity over complexity. By reducing visual clutter and organizing data into structured sections, the prototype demonstrates how analytics can be easier to interpret when presented in a more intentional way.

Limitations

Because Trulytics is a prototype, it does not connect to Instagram’s live API or update in real time. While exported data provided useful structural information, interaction metrics had to be collected manually rather than automatically retrieved. This limits scalability and automation.

Despite these limitations, the project still demonstrates how combining real data with clearer design can improve how users interpret engagement and performance metrics. AI Tool Usage

AI tools were used as a support tool to improve efficiency and assist with technical problem-solving. I used AI to help troubleshoot more complex JavaScript and understand how certain code structures worked when building interactive chart features.

AI also helped speed up repetitive tasks, such as duplicating HTML page structures and making consistent wording updates across multiple files. This allowed me to focus more on design decisions and how analytics are presented rather than repeating the same edits manually.

Additionally, AI helped summarize scholarly sources so I could quickly identify key concepts most relevant to my research question. AI functioned primarily as a time-saving and problem-solving tool, while all design decisions and interpretations were completed independently.

Why Methods Matter

Understanding how data is collected, structured, and visualized helps reveal how social media platforms shape user perception. Analytics are not neutral; they influence how individuals evaluate success, visibility, and audience response. By combining real data with alternative presentation methods, Trulytics demonstrates how design and analytics work together to influence interpretation.
